# Architecture Context

This repository is part of a three-system architecture.

openclashd-v2
- governance kernel
- proposal lifecycle
- extension lifecycle
- receipts and knowledge persistence

clashd27
- discovery engine
- signal generation
- observatory cube visualization

Jeeves
- human approval cockpit
- proposal queue
- decision history
- knowledge graph interface

System principle:
Agents propose capabilities.
Humans approve them.
The kernel executes bounded actions.
Results become knowledge.
